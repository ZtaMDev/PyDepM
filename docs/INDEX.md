## 🔑 Key Concepts

### Dependency Resolution

PyDepm uses a custom resolver (not pip) that:

- Parses requirements using PEP 508
- Fetches package info directly from PyPI
- Uses backtracking to find compatible versions
- Detects and reports conflicts
- [Learn more →](ARCHITECTURE.md#dependency-resolution)

### Lock Files

Generated by `pydep lock`, lock files:

- Record exact resolved versions and hashes
- Enable reproducible installs (10-50x faster)
- Validate package integrity (SHA256)
- Support both TOML and JSON formats
- [Learn more →](LOCK_FILE_FORMAT.md)

### Dependency Groups

Organize dependencies by purpose:

- `dependencies` - Core project dependencies (required)
- `dev` - Development and testing tools
- `docs` - Documentation generation
- `test` - Testing-specific tools
- Custom groups for specific needs
- [Learn more →](CONFIGURATION.md#optional-dependency-groups)

### Environment Management

PyDepm manages Python environments:

- Virtual environments (venv)
- Conda environments
- Global Python
- Automatic creation and activation
- [Learn more →](CLI_REFERENCE.md#3-environment-management)

## 💡 Quick Reference

### Most Common Commands

```bash
# Initialize project
pydep init myproject

# Add dependencies
pydep add requests
pydep add pytest --group dev

# Install dependencies
pydep install

# Generate lock file
pydep lock

# Update dependencies
pydep update --all

# Run scripts
pydep run test

# Check for issues
pydep audit --security

# Build and publish
pydep build
pydep publish
```

### Common Configurations

```toml
# Minimal project
[project]
name = "mylib"
version = "0.1.0"
requires-python = ">=3.11"
dependencies = ["requests"]

# Full-featured project
[project]
name = "myapp"
version = "1.0.0"
requires-python = ">=3.11"
dependencies = ["fastapi", "sqlalchemy"]

[project.optional-dependencies]
dev = ["pytest", "black", "ruff"]
docs = ["sphinx", "sphinx-rtd-theme"]

[tool.pydepm]
type = "app"  # or "module"
python-version = "3.11"

[tool.pydepm.scripts]
test = "pytest tests/"
dev = "uvicorn app:app --reload"
format = "black . && isort ."
```

## ❓ FAQ

**Q: Should I commit `pydepm.lock` to version control?**
A: Yes! Lock files ensure reproducibility. Always commit them.

**Q: How is PyDepm different from pip?**
A: PyDepm manages entire projects, not just installation. It includes building, publishing, dependency groups, lock files, and more.

**Q: Can I use PyDepm with existing Poetry/Pipenv projects?**
A: Yes! Use `pydep migrate` to convert, or manually set up with `pydep init`.

**Q: Is PyDepm faster than Poetry?**
A: Yes, especially with lock files (10-50x faster installs). Resolution is comparable or faster.

**Q: Can I use PyDepm with Docker/CI?**
A: Absolutely! See [CI/CD Integration](EXAMPLES.md#recipe-6-cicd-integration) in Examples.

**Q: What about Python 2 or older Python 3 versions?**
A: PyDepm requires Python 3.11+. Older versions aren't supported.
